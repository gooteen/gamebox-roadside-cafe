# gamebox-roadside-cafe
ФИО: Гутин Роман Сергеевич 

## Архитектура: 

1) GameController - главный хаб логики, сюда же вынесены все глобальные переменные, влияющие на геймплей.
Реализован как Singleton - этот паттерн в целом стараюсь использовать для всех одиночных объектов на сцене, имеющих много зависимостей (PlayerController, UserInterafaceController и т.д.).

2) Управление персонажем реализовано через мышь: обработка и типизация RayCast-хитов производится через CursorDetector, GameController получает результаты и передает их для обработки PlayerContoller.

3) Классы предметов, с которыми можно взаимодействовать, реализуют общий интерфейс IIController. 

4) Для всех спавнеров используется один класс Spawner с разделением логики через абстракции типов спавнеров и режимов работы.

5) Поведение Customers реализовано как State Machine c завязкой на вынесенные глобальные переменные. 

6) Частично (где уместно) применена event-driven архитектура: GameController генерирует события (изменение значений Денег и Загрязнения), а подписчики (UserInterafaceController, TreeController их обрабатывают).

7) Персонажи двигаются к целям, обходя препятствия. Это реализовано с помощью NavMeshPlus (см. зависимости).
